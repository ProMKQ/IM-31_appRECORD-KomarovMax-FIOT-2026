## Опис предметного середовища та бізнес-логіки

**Предметна галузь:** API для трекера завдань. Сервіс надає програмний інтерфейс для взаємодії з даними завдань, що дозволяє створювати клієнтські застосунки (веб, мобільні) для керування проєктами та особистими справами.

### Структура API

- **Автентифікація** - реєстрація та вхід користувача з отриманням JWT токена.
- **Завдання (Tasks)** - операції CRUD (Create, Read, Update, Delete) для завдань.
- **Користувачі (Users)** - оновлення профілю та керування користувачами (для адміністраторів).
- **Вкладення (Attachments)** - завантаження файлів до завдань.

### Сценарій взаємодії (бізнес-логіка)

1. **Гість** реєструє новий акаунт або входить у вже існуючий, отримуючи у відповідь JWT токен.
2. **Користувач**, використовуючи токен, може:
    - Створювати нові завдання.
    - Переглядати список своїх завдань.
    - Оновлювати статус або зміст існуючого завдання.
    - Видаляти завдання.
    - Завантажувати вкладення до завдання.
    - Оновлювати дані свого профілю.
3. **Адміністратор** має всі права користувача, а також може керувати акаунтами інших користувачів.

### Функціональні вимоги

- Реєстрація та автентифікація користувачів.
- Створення, читання, оновлення та видалення завдань (CRUD).
- Можливість змінювати статус завдання ('pending', 'in-progress', 'done').
- Завантаження файлів як укладень до завдань.
- Оновлення профілю користувача.
- Адміністратор має розширені права для керування користувачами.

### Нефункціональні вимоги

- API має бути RESTful.
- Відповіді сервера мають бути у форматі JSON.
- Час відповіді сервера - до 500 мс на стандартні запити.
- Захист ендпоінтів за допомогою JWT.
- Коректна обробка помилок та відповідні HTTP статуси.

### Стек технологій

- **Node.js** - середовище виконання для JavaScript на сервері.
- **Express.js** - фреймворк для створення вебсерверів та API.
- **Git + GitHub** - система контролю версій.

## Документація коду

Нижче наведено приклади реалізації основних ендпоінтів у файлі `backend/server.js`.

### Ініціалізація сервера та Middleware

Код для створення Express-застосунку та підключення middleware для обробки JSON.

```javascript
const express = require('express');
const app = express();
app.use(express.json());

let tasks = [
    { id: 1, title: 'Design database', status: 'Pending' },
    { id: 2, title: 'Build UI', status: 'In progress' }
];

app.listen(3000, () => {
    console.log('Server started on port 3000');
});
```

### Отримання списку завдань (GET /tasks)

Ендпоінт для отримання всіх завдань.

```javascript
app.get('/tasks', (req, res) => {
    res.json(tasks);
});
```

### Створення нового завдання (POST /tasks)

Ендпоіінт для додавання нового завдання. Дані передаються у тілі запиту.

```javascript
app.post('/tasks', (req, res) => {
    const newTask = {
        id: tasks.length + 1,
        title: req.body.title,
        status: req.body.status || 'Pending'
    };
    tasks.push(newTask);
    res.status(201).json(newTask);
});
```

### Оновлення завдання (PUT /tasks/:id)

Ендпоінт для оновлення існуючого завдання за його `id`.

```javascript
app.put('/tasks/:id', (req, res) => {
    const task = tasks.find(t => t.id === parseInt(req.params.id));
    if (!task) return res.status(404).send('Task not found');

    task.title = req.body.title || task.title;
    task.status = req.body.status || task.status;
    res.json(task);
});
```

### Видалення завдання (DELETE /tasks/:id)

Ендпоінт для видалення завдання за його `id`.

```javascript
app.delete('/tasks/:id', (req, res) => {
    tasks = tasks.filter(t => t.id !== parseInt(req.params.id));
    res.status(204).send();
});
```
